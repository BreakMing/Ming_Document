## 软件包是什么

像windows里的.exe后缀的安装包以及手机上.apk后缀的安装包，其本质都是软件包，

在以前开源软件只提供了源码文件，用户必须自己编译每个想在GNU/Linux上运行的软件，当Debian诞生时，这样的一个管理工具dpkg也就应运而生，可以用来管理deb后缀的“包“文件。从而著名的“package“的概念第一次出现在GNU/Linux系统中。

## 包的组成

- 二进制文件
- 库文件
- 配置文件
- 帮助文件












## 什么是包管理器

包管理器就像是手机上的应用商店一样，当我们需要往电脑上安装应用的时候就可以从它那里去下载我们需要的应用。

不同的Linux发行版也有不同的包管理器







### APT

apt update命令：更新软件包列表   相当于把一份最新的软件包列表下载到我们本地







debian系统包管理：dpkg  **apt**
arch系统包管理：pacman
gentoo系统包管理：emerge
slackware包管理：pkg
solus包管理：eopkg
redhat系统包管理：**dnf yum**
SUSE包管理：zypper

### 版本冲突

假设我们电脑上有多个项目要执行，但不同项目间可能是基于库的不同版本来写的，如果我们统一只有一个版本就会导致，有的项目可以使用这个版本的库，而有的项目会因为库版本的不同，导致用法或内容上产生了改变，导致项目不能执行
